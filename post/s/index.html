blog第一天——如何使用Gridea搭建博客

2025-11-24

准备

Github

在github上开一个仓库，一般以"用户名.github.io"为仓库名称。例如我的是zzzz-1111.github.io。注意了，是网址里面的那个用户名，而不是个人界面可以编辑的"Name"


Configuration使用的visibility保持默认的Public

Gridea

在Gridea官网上下载对应版本的安装包并安装。

Github token 获取

点击右上角，进入setting，然后再下滑到最下面的Developer setting，点击进去，来到

点击Generate new token (classic)

Note 随便写，在select scopes 里选上repo, workflow，然后把弹出来的token复制到其他地方保存（注意，只会显示一次，以后就看不到了）

Gridea 配置

来到“远程”

域名里填的是当时的仓库名，仓库名称也是填仓库名。
注意，只填仓库名，不要把前面的用户名也填进去了。就是不要填成zzzz-1111/zzzz-1111.github.oi形式 别问我怎么知道的

分支的话按照github上显示的选，我的是默认的main。仓库用户名就是github的用户名，那个网址里的用户名。
令牌是之前github的token，HTTP代理根据你自己的代理来填写。地址一般是127.0.0.1，端口看代理软件。

都填完之后，点左下角的检查远程连接，如果正常的话应该是这么显示的。

然后点击同步，就大功告成了。过一会，在仓库界面就可以看到很多由Gridea上传的文件了。

Github Page

在仓库/setting/Pages里检查Github Page的网址，打开来就可以了。
